# 异常页体系（403/404/500）

## 场景

异常页用于承接权限不足、页面不存在、服务异常等场景，避免用户进入空白页。

- 路由：`/examples/exception/403|404|500`
- 页面：`src/views/examples/exception/*.vue`

## 设计要点

- 403：强调“无权限”并提供返回入口。
- 404：强调“页面不存在”并提供首页/上一页。
- 500：强调“服务错误”并提供重试动作。

## 接入建议

1. 路由守卫中统一处理无权限跳转。
2. 未匹配路由统一重定向 `404`。
3. 全局错误边界捕获后导向 `500` 或展示错误块。
